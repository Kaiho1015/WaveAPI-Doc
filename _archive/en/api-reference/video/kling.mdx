Async processing mode -- submit a request and receive a task ID. Use the [Query Task Status](/en/api-reference/task/status) endpoint to retrieve generation results. Video links are valid for 24 hours.

<RequestExample>
```bash cURL
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v2-master",
    "action": "generate",
    "prompt": "一条锦鲤在清澈的池塘中游动，水面波光粼粼",
    "aspect_ratio": "16:9",
    "duration": 5,
    "extra": {
      "mode": "pro"
    }
  }'
```

    req.Header.Set("Content-Type", "application/json")

    resp, _ := http.DefaultClient.Do(req)
    defer resp.Body.Close()

## Available Models

| Model ID | Description |
|----------|-------------|
| `kling-video-o1` | Kling Video O1, latest flagship |
| `kling-v3` | Kling V3, supports 3-15 seconds, multi-shot, native audio |
| `kling-v2-master` | Kling V2 Master |
| `kling-v1-6` | Kling V1.6, supports camera control |
| `kling-v1` | Kling V1 |

## Common Parameters

<ParamField body="model" type="string" required>
  Model ID
</ParamField>

<ParamField body="action" type="string" default="generate">
  Operation type
</ParamField>

<ParamField body="prompt" type="string" required>
  Video description text, maximum 2500 characters
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  Aspect ratio: `16:9`, `9:16`, `1:1`

  Ignored for image-to-video; aspect ratio is determined by the input image
</ParamField>

<ParamField body="duration" type="integer" default="5">
  Video duration (seconds): `5`, `10`
</ParamField>

<ParamField body="negative_prompt" type="string">
  Content you do not want to appear, maximum 2500 characters. V1 / V1.6 only
</ParamField>

<ParamField body="image_urls" type="string[]">
  Reference image URL (first frame) for image-to-video. JPEG/PNG/WebP, up to 10MB
</ParamField>

<ParamField body="last_frame" type="string">
  Last frame image URL, requires `image_urls`
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook callback URL
</ParamField>

<ParamField body="extra" type="object">
  Additional parameters passed through to Kling (see model tabs)
</ParamField>

## Model-Specific Parameters

<Tabs>
  <Tab title="Kling V3">
    **duration range:** 3-15 seconds

    <ParamField body="extra.mode" type="string" default="std">
      Generation mode: `std` (720P) or `pro` (1080P)
    </ParamField>

    <ParamField body="extra.enable_audio" type="boolean" default="true">
      Native audio generation, enabled by default
    </ParamField>

    **Multi-shot mode:**

    Supports 2-6 shot segment generation using `shot_N_prompt` + `shot_N_duration` instead of `prompt` + `duration`:

    ```json
    "extra": {
      "shot_1_prompt": "特写：一杯咖啡在桌上冒着热气",
      "shot_1_duration": 5,
      "shot_2_prompt": "镜头拉远，展示整个咖啡馆的场景",
      "shot_2_duration": 5
    }
    ```

    Total duration across all shots must be within the 3-15 second range.
  </Tab>

  <Tab title="Kling V2 Master">
    <ParamField body="extra.mode" type="string" default="std">
      Generation mode: `std` (720P, faster) or `pro` (1080P, higher quality)
    </ParamField>

    <Note>
    V2 does not support `negative_prompt`, `cfg_scale`, or `camera_control`.
    </Note>
  </Tab>

  <Tab title="Kling V1.6">
    <ParamField body="extra.mode" type="string" default="std">
      Generation mode: `std` or `pro`
    </ParamField>

    <ParamField body="extra.cfg_scale" type="number" default="0.5">
      Creativity control, range 0-1. Higher values adhere more closely to the prompt
    </ParamField>

    <ParamField body="extra.camera_control" type="object">
      Camera motion control

      <Expandable title="Preset modes">
        `type` options: `down_back` (descend backward), `forward_up` (advance upward), `right_turn_forward` (right turn forward), `left_turn_forward` (left turn forward)
      </Expandable>

      <Expandable title="Custom mode">
        Set `type: "simple"` and control six-axis motion precisely via `config` (range -10 to 10):

        - `horizontal` — X translation, negative for left, positive for right
        - `vertical` — Y translation, negative for down, positive for up
        - `pan` — X rotation, negative for down tilt, positive for up tilt
        - `tilt` — Y rotation, negative for left turn, positive for right turn
        - `roll` — Z rotation, negative for counterclockwise, positive for clockwise
        - `zoom` — Focal length, negative for zoom in, positive for zoom out
      </Expandable>
    </ParamField>

    <ParamField body="extra.static_mask" type="string">
      Static mask image URL (PNG), defines areas that remain stationary. Image-to-video only
    </ParamField>

    <ParamField body="extra.dynamic_masks" type="array">
      Dynamic mask array (up to 6), defines areas that move along trajectories. Image-to-video only

      <Expandable title="Properties">
        <ParamField body="mask_url" type="string">
          Mask image URL (PNG with alpha)
        </ParamField>
        <ParamField body="trajectories" type="array">
          Trajectory coordinate point array, each point contains x and y coordinates, 2-77 points
        </ParamField>
      </Expandable>
    </ParamField>
  </Tab>

  <Tab title="Kling V1">
    <ParamField body="extra.mode" type="string" default="std">
      Generation mode: `std` or `pro`
    </ParamField>

    <ParamField body="extra.cfg_scale" type="number" default="0.5">
      Creativity control, range 0-1
    </ParamField>
  </Tab>
</Tabs>

## Generation Modes

| Mode | Parameters | Description |
|------|-----------|-------------|
| Text-to-video | `prompt` | Pure text generation |
| Image-to-video | `prompt` + `image_urls` (1 image) | First frame image + text description |
| First/Last frame | `image_urls` (1 image) + `last_frame` | Transition between start and end frames |

<Note>
For image-to-video, `aspect_ratio` is determined by the input image and does not need to be specified.
</Note>
