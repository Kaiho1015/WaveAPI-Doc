Async processing mode -- submit a request and receive a task ID. Use the [Query Task Status](/en/api-reference/task/status) endpoint to retrieve generation results.

<RequestExample>
```bash cURL
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "flux-kontext-pro",
    "prompt": "一只橘猫趴在窗台上晒太阳，水彩画风格",
    "size": "16:9"
  }'
```

```python Python
from openai import OpenAI

client = OpenAI(
    base_url="https://www.qingbo.dev/v1",
    api_key="YOUR_API_KEY"
)

response = client.images.generate(
    model="flux-kontext-pro",
    prompt="一只橘猫趴在窗台上晒太阳，水彩画风格",
    size="16:9"
)

print(response.data[0])
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
)

func main() {
    payload := map[string]interface{}{
        "model":  "flux-kontext-pro",
        "prompt": "一只橘猫趴在窗台上晒太阳，水彩画风格",
        "size":   "16:9",
    }

    body, _ := json.Marshal(payload)
    req, _ := http.NewRequest("POST", "https://www.qingbo.dev/v1/tasks", bytes.NewBuffer(body))
    req.Header.Set("Authorization", "Bearer YOUR_API_KEY")
    req.Header.Set("Content-Type", "application/json")

    resp, _ := http.DefaultClient.Do(req)
    defer resp.Body.Close()
    result, _ := io.ReadAll(resp.Body)
    fmt.Println(string(result))
}
```

```java Java
import java.net.http.*;
import java.net.URI;

public class Main {
    public static void main(String[] args) throws Exception {
        String payload = """
        {
          "model": "flux-kontext-pro",
          "prompt": "一只橘猫趴在窗台上晒太阳，水彩画风格",
          "size": "16:9"
        }
        """;

        HttpClient client = HttpClient.newHttpClient();
        HttpRequest request = HttpRequest.newBuilder()
            .uri(URI.create("https://www.qingbo.dev/v1/tasks"))
            .header("Authorization", "Bearer YOUR_API_KEY")
            .header("Content-Type", "application/json")
            .POST(HttpRequest.BodyPublishers.ofString(payload))
            .build();

        HttpResponse<String> response = client.send(request,
            HttpResponse.BodyHandlers.ofString());
        System.out.println(response.body());
    }
}
```

## Available Models

| Model ID | Description |
|----------|-------------|
| `flux-kontext-pro` | FLUX Kontext Pro, excels at image editing and style transfer |
| `flux-kontext-max` | FLUX Kontext Max, enhanced version |
| `flux-2-pro` | FLUX 2.0 Pro, general-purpose image generation |
| `flux-2-dev` | FLUX 2.0 Dev, development version |

## Common Parameters

<ParamField body="model" type="string" required>
  Model ID, see the model list above
</ParamField>

<ParamField body="prompt" type="string" required>
  Image description text, maximum 1000 characters
</ParamField>

<ParamField body="image_urls" type="array">
  Reference image URL array
</ParamField>

## Model-Specific Parameters

<Tabs>
  <Tab title="FLUX Kontext">
    Applies to `flux-kontext-pro` and `flux-kontext-max`

    <ParamField body="size" type="string" default="match_input_image">
      Image size

      Supported: `match_input_image`, `auto`, `1:1`, `4:3`, `3:4`, `16:9`, `9:16`, `3:2`, `2:3`, `21:9`, `9:21`
    </ParamField>

    <ParamField body="response_format" type="string" default="png">
      Output format: `png`, `jpg`
    </ParamField>

    <ParamField body="safety_tolerance" type="integer" default="2">
      Content safety level, range 0-6. Higher values are more permissive
    </ParamField>

    <ParamField body="prompt_upsampling" type="boolean" default="false">
      Enable prompt optimization enhancement
    </ParamField>

    <Note>
    The Kontext series focuses on **image editing**. `image_urls` accepts a maximum of 1 reference image.
    </Note>
  </Tab>

  <Tab title="FLUX 2.0">
    Applies to `flux-2-pro` and `flux-2-dev`

    <ParamField body="size" type="string" default="1:1">
      Image aspect ratio

      Supported: `1:1`, `2:3`, `3:2`, `3:4`, `4:3`, `4:5`, `5:4`, `9:16`, `16:9`, `21:9`
    </ParamField>

    <Note>
    The FLUX 2.0 series focuses on **general-purpose image generation**. `image_urls` can be used for image-to-image reference.
    </Note>
  </Tab>
</Tabs>

## Response

Returns a task ID. Use [Query Task Status](/en/api-reference/task/status) to retrieve the final image result.

<ResponseField name="code" type="string">
  Status code, `"success"` on success
</ResponseField>

<ResponseField name="data" type="string">
  Task ID, formatted as `task-wave{timestamp}{userId}{random}`
</ResponseField>
