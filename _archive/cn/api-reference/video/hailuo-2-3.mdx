## 认证方式

在请求头中使用 API Key 进行身份验证。您可以在 [API Key 管理](https://qingbo.dev/dashboard/keys) 页面获取您的密钥。

```http theme={null}
Authorization: Bearer YOUR_API_KEY
```

## 请求参数

<ParamField body="model" type="string" required>
  模型名称，固定为 `minimax-hailuo-2.3`
</ParamField>

<ParamField body="prompt" type="string" required>
  视频生成的提示词。支持使用 `[命令]` 语法控制运镜。
</ParamField>

<ParamField body="duration" type="integer" default="6">
  视频时长。可选值：`6` 或 `10`。注意 `1080p` 分辨率下仅支持 `6` 秒时长。
</ParamField>

<ParamField body="resolution" type="string" default="768p">
  视频分辨率。
  * `768p`: 1344x768 (横屏) / 768x1344 (竖屏)
  * `1080p`: 1920x1080 (横屏) / 1080x1920 (竖屏)
</ParamField>

<ParamField body="first_frame" type="string">
  首帧图片。支持 URL 或 Base64 编码字符串。提供此参数后，模型将基于该图片生成视频。
</ParamField>

<ParamField body="prompt_optimizer" type="boolean" default="true">
  是否开启提示词优化器。
</ParamField>

<ParamField body="fast_pretreatment" type="boolean" default="false">
  是否开启快速预处理。
</ParamField>

<ParamField body="watermark" type="boolean" default="false">
  是否在生成的视频中包含水印。
</ParamField>

### 分辨率与时长矩阵

| 分辨率 | 时长 (s) | 备注 |
| :--- | :--- | :--- |
| `768p` | 6 / 10 | 默认分辨率 |
| `1080p` | 6 | 高清模式 |

### 运镜控制命令

您可以在 `prompt` 中加入以下命令来精确控制相机移动：

| 命令 | 说明 | 命令 | 说明 |
| :--- | :--- | :--- | :--- |
| `[左移]` | 相机向左移动 | `[右移]` | 相机向右移动 |
| `[左摇]` | 相机向左水平旋转 | `[右摇]` | 相机向右水平旋转 |
| `[推进]` | 相机向前推进 | `[拉远]` | 相机向后拉远 |
| `[上升]` | 相机向上提升 | `[下降]` | 相机向下降低 |
| `[上摇]` | 相机向上垂直旋转 | `[下摇]` | 相机向下垂直旋转 |
| `[变焦推近]` | 镜头光学变焦放大 | `[变焦拉远]` | 镜头光学变焦缩小 |
| `[晃动]` | 相机随机晃动 | `[跟随]` | 相机跟随主体移动 |
| `[固定]` | 固定机位 | | |

## 注意事项

1. **异步处理**: 视频生成通常需要数分钟。请记录返回的 `data` (任务 ID) 并轮询状态接口。
2. **错误处理**:
   * `invalid_request`: 请求参数无效。
   * `authentication_error`: 身份验证失败，请检查您的 API 密钥。
   * `payment_required`: 账户余额不足，请充值后再试。
   * `quota_not_enough`: 用户额度不足。
   * `rate_limit_error`: 请求过于频繁，请稍后再试。
   * `server_error`: 服务器内部错误，请稍后重试。
3. **参数限制**: `1080p` 不支持 10 秒时长，请求时请务必注意参数组合。
