OpenAI 在 2025 年 9-10 月发布的新一代视频生成模型。

相比 Sora 1,**Sora 2 Preview** 在以下三个维度有显著提升:

- **物理真实感** — 碰撞、流体、刚体动力学、布料与重力反馈更接近真实世界,对杂技、运动、跌落、液体倾倒等高难度动作的稳定性大幅改善;
- **角色一致性** — 跨镜头与跨场景的身份保持更强,主体面部、服装、配饰在长时镜头里漂移更小;
- **原生有声** — 直接生成与画面同步的背景音乐、人声轮廓与环境音,不再需要单独走 TTS / 音乐生成管线再做混音。

支持**文生视频(T2V)** 与**图生视频(I2V,单图首帧引导)** 两种模式,**不支持**多图参考(R2V)、视频续写(extend)与视频编辑(edit)—— 这些场景请改用 [HappyHorse 系列](/cn/api-reference/video/happyhorse) 或其他视频模型。

时长固定为 **4 / 8 / 12 秒**,宽高比仅支持 **16:9**(横版)与 **9:16**(竖版),**不支持方形**。生成完成后返回的视频与封面 URL 由 OpenAI 侧签发,**仅 72 小时有效**,请在收到终态后尽快转存。

## Sora 2 vs Sora 1 改进点

| 维度 | Sora 1 | Sora 2 Preview |
|---|---|---|
| 物理真实感 | 一般,易出现穿模与重力错位 | **显著提升**,刚体/流体/布料更稳 |
| 角色一致性 | 跨镜头主体易漂移 | **跨镜头身份保持更强** |
| 音频 | 无,需后期单独混音 | **原生有声**,与画面同步生成 |
| 镜头语言理解 | 基本主体 + 动作 | dolly / push / handheld / aerial 等指令更精准 |
| 时长 | 多档可选 | 固定 `4` / `8` / `12` 秒 |
| 宽高比 | 多比例 | **仅 `16:9` / `9:16`** |
| 链接有效期 | — | **72 小时**,需自行转存 |

## 模式速查

| 传入字段 | 路由模式 | 说明 |
|---|---|---|
| 仅 `prompt` | **T2V**(文生视频) | 画面比例由 `aspect_ratio` 控制(支持 `landscape` / `portrait` 别名) |
| `+ image_urls`(单元素) | **I2V**(图生视频) | **只取第一张**,且 `aspect_ratio` 被图片本身宽高比覆盖 |

<Warning>
**不支持** R2V(多图参考)、extend(视频续写)、edit(视频编辑)、remix。这些场景请走 [HappyHorse](/cn/api-reference/video/happyhorse) 或其他渠道。
</Warning>

## 调用示例

<CodeGroup>

```bash 文生视频(最简)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sora-2-preview",
    "prompt": "A cat playing piano in a jazz club, cinematic lighting, warm tone, smooth dolly-in",
    "aspect_ratio": "16:9",
    "duration": 8,
    "resolution": "720p"
  }'
```

```bash 文生视频(竖版别名)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sora-2-preview",
    "prompt": "A skateboarder jumps over a fountain at golden hour, slow motion, ambient city sound",
    "aspect_ratio": "portrait",
    "duration": 12,
    "resolution": "720p",
    "seed": 42
  }'
```

```bash 图生视频(I2V 单图)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sora-2-preview",
    "action": "image2video",
    "prompt": "Camera slowly pushes in, subject turns head and smiles, soft natural light",
    "image_urls": ["https://cdn.example.com/portrait.jpg"],
    "duration": 4,
    "resolution": "720p"
  }'
```

```bash 带 Webhook 回调
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "sora-2-preview",
    "prompt": "Aerial shot of a coastal cliff at sunset, waves crashing below",
    "aspect_ratio": "16:9",
    "duration": 8,
    "resolution": "720p",
    "callback_url": "https://your-app.example.com/wave/callback",
    "callback_events": ["succeeded", "failed"]
  }'
```

</CodeGroup>

<ResponseExample>
```json 提交成功
{
  "task_id": "task-wave1775285160b950328499",
  "model": "sora-2-preview",
  "action": "generate",
  "status": "queued",
  "created_at": 1775285160040,
  "progress": 0
}
```
</ResponseExample>

提交后用 [`GET /v1/tasks/{task_id}`](/cn/api-reference/task/status) 轮询状态,详见 [任务系统](/cn/docs/task-system)。终态成功时,响应里的 `_urls` 字段会包含可下载的视频与封面链接。

**典型错误响应:**

```json 不支持的宽高比
{
  "error": {
    "code": "invalid_aspect_ratio",
    "message": "Sora 2 Preview only supports 16:9 (landscape) and 9:16 (portrait)."
  }
}
```

```json 不支持的时长
{
  "error": {
    "code": "invalid_duration",
    "message": "duration must be one of [4, 8, 12]."
  }
}
```

## 通用参数

<ParamField body="model" type="string" required>
  固定为 `sora-2-preview`
</ParamField>

<ParamField body="action" type="string" default="generate">
  操作类型,**通常无需显式传**,后端按是否提供 `image_urls` 自动路由。可选值:
  - `generate` — 文生视频(T2V)
  - `image2video` — 图生视频(I2V),需配合 `image_urls`
</ParamField>

<ParamField body="prompt" type="string" required>
  视频描述文本。建议英文或中英混排,描述**主体、动作、镜头运动、光影与情绪**。Sora 2 对镜头语言(dolly-in / push / handheld / aerial 等)与时间状态(slow motion / time-lapse)的理解显著强于 Sora 1
</ParamField>

<ParamField body="n" type="integer" default="1">
  一次提交生成的视频数量。当前 Preview 通道建议保持 `1`,提高并发请改用多个独立任务
</ParamField>

<ParamField body="seed" type="integer" default="-1">
  随机种子,`-1` 为随机;固定值在相同 prompt 下可复现相似结果(注意视频生成的种子稳定性弱于图像)
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  画面宽高比,**仅 T2V 模式生效**;I2V 模式下会被图片本身的宽高比强制覆盖。可选值:
  - `16:9` — 横版宽屏(也接受别名 `landscape`)
  - `9:16` — 竖版长屏(也接受别名 `portrait`)

  <Warning>
  Sora 2 Preview **不支持** `1:1` 方形,也不支持 `4:3` / `3:4` / `21:9` 等其他比例,传入会被渠道直接拒绝。
  </Warning>
</ParamField>

<ParamField body="resolution" type="string" default="720p">
  输出分辨率。当前 Preview 通道仅支持:
  - `720p`
</ParamField>

<ParamField body="duration" type="integer" default="4">
  视频时长(秒),**固定枚举**,不接受任意整数。可选值:
  - `4`
  - `8`
  - `12`
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图片 URL 数组,触发 **I2V** 模式:
  - **只取第一张** — 即便传入多张,后端也只读取 `image_urls[0]` 作为首帧引导,其余元素会被静默忽略
  - **图片宽高比覆盖请求** — I2V 时实际输出的宽高比强制跟随该图片本身,请求中的 `aspect_ratio` 会被忽略
  - 公网可下载链接,JPG / PNG / WEBP,暂不支持 base64
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [回调机制](/cn/docs/sync-async#异步任务-webhook)
</ParamField>

<ParamField body="callback_events" type="string[]">
  订阅的回调事件类型,默认仅在终态(`succeeded` / `failed`)推送。可选值见 [任务系统](/cn/docs/task-system)
</ParamField>

### Prompt 写作建议

Sora 2 对镜头语言与时间状态的理解显著强于上一代,推荐在 prompt 里显式给出:

| 维度 | 推荐表达 |
|---|---|
| 镜头运动 | `dolly-in` / `push-in` / `pull-out` / `handheld` / `aerial` / `tracking shot` |
| 时间状态 | `slow motion` / `time-lapse` / `real-time` |
| 光影 | `golden hour` / `cinematic lighting` / `neon backlight` / `soft natural light` |
| 情绪 / 风格 | `noir` / `documentary` / `studio ghibli style`(注意版权)/ `vintage 16mm film grain` |
| 主体动作 | 越具体越好 — "turns head and smiles" 优于 "the person moves" |

中英混排可用,但**核心动作动词**用英文通常更稳定。

## 模型特定参数

Sora 2 Preview 兼容 OpenAI 官方语义,在通用参数之外提供下列**别名/兼容写法**:

<ParamField body="aspect_ratio: landscape" type="alias">
  等价于 `aspect_ratio: "16:9"`,后端会归一化。如果你直接复用 OpenAI 官方 SDK 的代码,**无需改动**即可对接。
</ParamField>

<ParamField body="aspect_ratio: portrait" type="alias">
  等价于 `aspect_ratio: "9:16"`,后端会归一化。
</ParamField>

<Note>
两种写法在同一字段中**任选其一**即可,不要同时传 `16:9` 和 `landscape` 这类冗余值;若两次请求里别名与标准值混用,后端最终落库的都是标准 `16:9` / `9:16`。
</Note>

## 注意事项

<Warning>
**I2V 时图片宽高比强制覆盖请求** — 一旦传入 `image_urls`,输出视频的宽高比将完全由该图片决定;请求体里的 `aspect_ratio` 会被静默忽略。如需特定输出比例,请**提前对参考图做裁切**(例如先裁成 16:9 再上传),不要指望 API 侧帮你处理。
</Warning>

<Note>
**只取第一张参考图** — Sora 2 Preview 当前**不支持多图参考(R2V)**,`image_urls` 数组中的第二张及之后会被忽略,也不会报错。需要多图风格 / 角色一致性请改用 [HappyHorse R2V](/cn/api-reference/video/happyhorse) 或 [Hailuo 系列](/cn/api-reference/video/hailuo)。
</Note>

<Note>
**aspect_ratio 接受别名** — 如果你的代码沿用 OpenAI 官方 SDK 写法传入 `landscape` / `portrait`,无需改动,渠道侧会自动归一化为 `16:9` / `9:16`,响应里也会回填标准值。
</Note>

<Warning>
**不支持方形 / 自定义比例** — 当前预览版仅 `16:9` / `9:16` 两种比例,传入 `1:1` / `4:3` / `3:4` / `21:9` 等会被直接拒绝(`400 invalid_aspect_ratio`)。
</Warning>

<Warning>
**视频链接 72 小时有效** — 任务完成后返回的 `_urls` 字段(视频与封面)由 OpenAI 侧签发,**有效期为 72 小时**。请在收到 `succeeded` 后尽快下载并转存到自己的对象存储,避免链接过期导致 C 端用户 404。建议把"任务终态 → 立刻拉取 → 转存到自有 CDN → 替换业务侧 URL"做成同一个 worker 流程。
</Warning>

<Note>
**原生有声但不可控** — Sora 2 Preview 的音轨与画面同步生成,**当前无法关闭、不可单独替换、也不接受外部 BGM / 配音参数**。如果业务侧需要静音视频或自定义音轨,请在收到产物后用 ffmpeg 等工具二次处理。
</Note>

<Note>
**Preview 通道不保证稳定性** — Preview 模型可能随时调整价格、限流、可用区或下线。生产环境长期依赖前请同时灰度备选模型(如 HappyHorse、Hailuo)以便快速切换。
</Note>

## 资源限制

| 项目 | 限制 |
|---|---|
| 参考图片(I2V) | **仅取第一张**,JPG / PNG / WEBP,公网可下载,暂不支持 base64 |
| 时长 | 仅 `4` / `8` / `12` 秒,**不支持任意整数** |
| 宽高比 | 仅 `16:9` / `9:16`,**不支持方形** 与其他比例 |
| 分辨率 | 仅 `720p` |
| 单次提交数量 | 建议 `n = 1` |
| 输出格式 | MP4,**原生带音轨** |
| 输出链接有效期 | **72 小时**,过期需重新生成 |
