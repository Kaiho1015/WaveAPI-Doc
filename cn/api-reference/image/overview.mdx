---

WaveAPI 支持多家图像生成模型,**统一通过异步任务接口** `/v1/tasks` 调用。图像生成通常 1-30 秒完成,提交后返回 `task_id`,通过轮询或 Webhook 回调获取结果。

## 请求端点

```
POST https://www.qingbo.dev/v1/tasks
```

异步流程详见 [任务系统](/cn/docs/task-system) — Submit → Status → Result + 可选 webhook。

## 通用参数(全系列共有)

> 各模型实际支持范围不同,**详见各 vendor 文档**。

<ParamField body="model" type="string" required>
  模型 ID(group_name),从上面的 vendor 文档选择
</ParamField>

<ParamField body="action" type="string" default="generate">
  操作类型,可选值取决于模型,常见:
  - `generate` — 文生图(默认)
  - `image2image` — 图生图(需配合 `image_urls`)
  - `edit` — 图像编辑(局部重绘 / 光影修改)
  - `reference` — 多图参考融合
  - `inpaint` — 局部重绘(需配合 `mask_url`,GPT-Image 系列)
  - `group` — 组图生成(主题相关一组)
  - `interactive_edit` — 交互式编辑(Wan Image 独家,带 bbox 框选)
</ParamField>

<ParamField body="prompt" type="string" required>
  图像描述文本,支持中英文
</ParamField>

<ParamField body="n" type="integer" default="1">
  生成数量(部分 vendor 限制单次张数)
</ParamField>

<ParamField body="seed" type="integer" default="-1">
  随机种子,`-1` 为随机;固定值可复现相似结果
</ParamField>

<ParamField body="aspect_ratio" type="string">
  画面宽高比,如 `16:9` / `9:16` / `1:1`,部分 vendor 支持 `auto`(智能选择)
</ParamField>

<ParamField body="resolution" type="string">
  输出分辨率,如 `1K` / `2K` / `4K`,各 vendor 支持范围不同
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图片 URL 数组(图生图 / 多图参考用)
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [任务系统](/cn/docs/task-system#webhook)
</ParamField>

## 提交响应示例

```json
{
  "task_id": "task-wave1775285160b950328499",
  "model": "doubao-seedream-4.5",
  "action": "generate",
  "status": "queued",
  "created_at": 1775285160040,
  "progress": 0
}
```

拿到 `task_id` 后调 [`GET /v1/tasks/{task_id}`](/cn/api-reference/task/status) 轮询,直到 `status = completed` 取图片 URL。

## 模式速查

| 模式 | 触发字段 | 典型 vendor |
|---|---|---|
| **文生图(T2I)** | 仅 `prompt` | 全部 |
| **图生图 / 多图参考** | `+ image_urls`(单图或多图) | Seedream / Gemini / GPT-Image / Wan / Qwen |
| **图像编辑(局部重绘 / 光影修改)** | `+ image_urls + action: edit` | Seedream 4.0/4.5 / Gemini / GPT-Image / Wan |
| **局部重绘(带 mask)** | `+ image_urls + mask_url + action: inpaint` | GPT-Image 系列 |
| **组图生成(主题相关一组)** | `sequential_image_generation: auto` 或 `enable_sequential: true` | Seedream / Wan |
| **交互式编辑(bbox 框选)** | `+ image_urls + bbox_list` | Wan Image |
| **搜索增强出图** | `google_search: true` 或 `google_image_search: true` | Gemini 3.1 Flash |

<Note>
并非所有模型都支持全部模式 — 各 vendor 实际支持的 `action` 列表 + 字段范围请查看对应文档。后端会校验请求字段是否在 vendor 声明的能力范围内,不在范围会返回错误。
</Note>

## 字段命名规范

- **媒体引用统一复数** — 一律 `image_urls`,即使单张也用单元素数组 `["one.jpg"]`
- **画面比例统一** `aspect_ratio` — vendor 内部用 `size` / `ratio` 等是实现细节,无需感知
- **分辨率统一** `resolution` — vendor 内部用 `quality` 等是实现细节
- **遮罩字段** `mask_url`(GPT-Image 系列局部重绘用)
