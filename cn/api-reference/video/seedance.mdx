豆包 Seedance 视频生成系列,覆盖 **旗舰加速 (Pro Fast) → 旗舰高质量 (Pro Quality) → 有声 1.5 → 多模态 2.0** 全档位。

| 系列分代 | 定位 |
|---|---|
| **1.0 Pro Fast** | 旗舰加速档,质量速度均衡,出片速度约 Pro Quality 3 倍 |
| **1.0 Pro Quality** | 旗舰高质量档,1080P 多镜头叙事,适合最终交付 |
| **1.5 Pro** | 新一代有声视频,音视频联合生成,多语言对白 + 口型同步 |
| **2.0** | 多模态旗舰,文+图(≤9)+视频(≤3)+音频(≤3)统一架构,`@image1`/`@video2`/`@audio3` 引用语法 |
| **2.0 Face** | 2.0 强化人脸/身份参考,数字人短剧、口播 |
| **2.0 Fast / Fast Face** | 2.0 加速档,无 1080P,换更快出片 + 更低成本 |

## 模式路由

整个系列共用字段路由约定 — 后端按传入媒体字段自动决定 action,通常无需显式传 `action`。

| 传入字段 | 路由模式 | 说明 |
|---|---|---|
| 仅 `prompt` | **generate**(T2V) | 文生视频,画面比例由 `aspect_ratio` 控制 |
| `+ first_frame_image` | **image2video**(I2V) | 首帧驱动,跟随首帧比例 |
| `+ first_frame_image + last_frame_image` | **first_last_frame** | 首帧 + 尾帧约束插值 |
| `+ image_urls` | **reference** | 多图角色 / 风格一致性 |
| `+ video_urls`(2.0) | **reference_video** | 视频片段参考(2.0 独有) |
| `+ audio_urls`(2.0) | **reference_audio** | 音频驱动(2.0 独有) |

<Warning>
**2.0 多素材引用语法** — `prompt` 中可用 `@image1` / `@video2` / `@audio3` 等占位符引用 `image_urls[0]` / `video_urls[1]` / `audio_urls[2]`,索引从 1 开始。
</Warning>

## 调用示例

<CodeGroup>

```bash 文生视频(Pro Fast)

  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-1.0-pro-fast",
    "prompt": "一只柴犬穿宇航服在月球上行走,电影感光影",
    "duration": 5,
    "resolution": "720p",
    "aspect_ratio": "16:9"
  }'
```

## 通用参数

<ParamField body="model" type="string" required>
  模型 ID,见上方"可用模型"表
</ParamField>

<ParamField body="action" type="string" default="generate">
  操作类型,**通常无需显式传**,后端按媒体字段自动路由。可选值:
  - `generate` — 文生视频(T2V)
  - `image2video` — 图生视频(I2V),需配合 `first_frame_image`
  - `first_last_frame` — 首尾帧约束,需配合 `first_frame_image` + `last_frame_image`
  - `reference` — 参考图驱动,需配合 `image_urls`
  - `reference_video` — 视频片段参考(仅 2.0 系列),需配合 `video_urls`
  - `reference_audio` — 音频驱动(仅 2.0 系列),需配合 `audio_urls`
</ParamField>

<ParamField body="prompt" type="string">
  视频描述文本。**T2V 必填**;其他模式可选作引导。2.0 系列支持 `@image1` / `@video2` / `@audio3` 引用语法
</ParamField>

<ParamField body="n" type="integer" default="1">
  单次任务并行生成的视频数量
</ParamField>

<ParamField body="seed" type="integer" default="-1">
  随机种子,`-1` 为随机;固定值可复现相似结果
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  画面宽高比。可选值:
  - `16:9` — 横版宽屏
  - `9:16` — 竖版长屏
  - `1:1` — 正方形
  - `4:3` — 横版
  - `3:4` — 竖版
  - `21:9` — 超宽屏
  - `adaptive` — 自适应(**仅 2.0 系列**,跟随参考素材)
</ParamField>

<ParamField body="resolution" type="string" default="720p">
  输出分辨率,可选值因模型而异:
  - `480p`
  - `720p`
  - `1080p`(Lite-i2v / 2.0-fast / 2.0-fast-face 不支持)
</ParamField>

<ParamField body="duration" type="integer" default="5">
  视频时长(秒)。范围因系列而异:
  - 1.0 系列:2-12
  - 1.5 Pro:4-12
  - 2.0 系列:4-15
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图 URL 数组,触发 **reference** 模式。1.0 / 1.5 支持 1-9 张,2.0 支持 ≤9 张
</ParamField>

<ParamField body="first_frame_image" type="string">
  视频首帧图 URL,触发 **image2video** 或 **first_last_frame** 模式
</ParamField>

<ParamField body="last_frame_image" type="string">
  视频尾帧图 URL,与 `first_frame_image` 同传触发 **first_last_frame** 模式
</ParamField>

<ParamField body="video_urls" type="string[]">
  参考视频 URL 数组(**仅 2.0 系列**),≤3 段,触发 **reference_video** 模式。**计费按上方"视频参考子价"表替换基础价**(显著低于纯生成)
</ParamField>

<ParamField body="audio_urls" type="string[]">
  参考音频 URL 数组(**仅 2.0 系列**),≤3 条,触发 **reference_audio** 模式
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [回调机制](/cn/docs/sync-async#异步任务-webhook)
</ParamField>

<ParamField body="callback_events" type="string[]">
  订阅的回调事件类型,默认仅终态(`completed` / `failed`)
</ParamField>

## 模型特定参数

<Tabs>

<Tab title="2.0 Fast / 2.0 Fast Face">

参数与 2.0 / 2.0 Face 相同(`generate_audio` / `tools` / `image_with_roles` / `return_last_frame`)。

**差异**:
- 仅 480P / 720P,**无 1080P**
- 加速档,质量略降换更快出片 + 更低成本
- Fast Face 子型保留人脸一致性能力
- 时长 4-15 秒

**典型用途**:
- 数字人短视频批量生产
- 社媒口播运营
- 素材快速迭代

</Tab>

<Tab title="2.0 / 2.0 Face">

<ParamField body="generate_audio" type="boolean" default="false">
  是否生成音频(有声视频)。2.0 系列字段名为 `generate_audio`
</ParamField>

<ParamField body="tools" type="array">
  工具列表,例:`[{"type": "web_search"}]` 启用网络检索增强
</ParamField>

<ParamField body="image_with_roles" type="array">
  带角色标记的参考图,优先级高于 `image_urls` + `first_frame_image` / `last_frame_image`。元素结构:
  - `url` — 图片 URL
  - `role` — `first_frame` / `last_frame`
</ParamField>

<ParamField body="return_last_frame" type="boolean" default="false">
  是否额外返回视频尾帧图,用于连续生成续集
</ParamField>

**特性(2.0)**:
- 多模态统一架构:文 + 图(≤9)+ 视频(≤3)+ 音频(≤3)任意组合
- `@image1` / `@video2` / `@audio3` prompt 引用语法
- `aspect_ratio` 支持 `adaptive`
- 时长 4-15 秒

**2.0 Face 额外**:
- 强化人脸 / 身份一致性,五官 + 肤色多镜头稳定
- 适合数字人短剧 / 口播

</Tab>

<Tab title="1.5 Pro">

<ParamField body="audio" type="boolean" default="false">
  是否生成音频(有声视频)。1.5 系列字段名为 `audio`,**与 2.0 的 `generate_audio` 不同**
</ParamField>

<ParamField body="camerafixed" type="boolean" default="false">
  是否固定摄像头
</ParamField>

**特性**:
- 音视频联合生成,口型同步
- 多语言对白:中 / 英 / 日 / 韩 / 西
- 方言:四川话 / 粤语
- 时长 4-12 秒

</Tab>

<Tab title="1.0 Pro Quality">

无模型特定参数,仅使用通用参数。

**特性**:
- 旗舰高质量档,极致画面与运动表现
- 1080P 多镜头叙事,适合最终交付
- 全 action 支持(generate / image2video / first_last_frame / reference)
- 时长 2-12 秒

</Tab>

<Tab title="1.0 Pro Fast">

无模型特定参数,仅使用通用参数。

**特性**:
- 旗舰加速档,出片速度约 Pro Quality 3 倍
- 480P / 720P / 1080P 全分辨率
- 时长 2-12 秒
- 不支持 `first_last_frame`

</Tab>

</Tabs>

## 资源限制

| 项目 | 限制 |
|---|---|
| 参考图(`image_urls`) | 最多 9 张,单张 ≤ 30MB,JPG / PNG / WEBP |
| 首尾帧(`first_frame_image` / `last_frame_image`) | 单张 ≤ 30MB,JPG / PNG / WEBP |
| 参考视频(`video_urls`,仅 2.0) | 最多 3 段,MP4 / MOV,≤ 100MB / 段 |
| 参考音频(`audio_urls`,仅 2.0) | 最多 3 条,WAV / MP3,≤ 15MB / 条 |
| 输出 | MP4,链接 24 小时有效 |
