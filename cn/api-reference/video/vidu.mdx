生数科技 Vidu Q3 系列视频生成模型,**参考生视频(R2V)** 是看家本事 — 接受 1-7 张公网图作为主体 / 风格参考,prompt 只负责描述动作与镜头,人物外观由参考图决定,适合 **角色一致性 / 风格延续 / 多机位切换** 三类场景。

四档定位:

- **`viduq3-pro`** — 旗舰档,**按 `image_urls` 数量自动路由** 文生 / 图生 / 首尾帧,默认开启音频直出(对白 + 音效)
- **`viduq3-mix`** — 综合画质王,支持 **多段参考视频 / 图片混剪** 生成连贯视频,带音频联合生成,720P / 1080P 双档,适合短剧分镜重组、广告素材二创
- **`viduq3`** — 标准档,日常默认首选,镜头切换更聪明,540p / 720p / 1080p,3-16 秒
- **`viduq3-turbo`** — 轻量快速档,生成最快、单价最低,质量略低于标准档,适合预览 / 批量 / A/B 测试

按 **分辨率 × 时长** 计费,**每秒单价**,1-16 秒任意整数。

## 调用示例

<CodeGroup>

```bash 文生视频(Pro)

  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "viduq3-pro",
    "prompt": "黄昏的海滩,女孩迎着夕阳奔跑,长发飞扬,电影感光影",
    "duration": 5,
    "resolution": "1080p",
    "aspect_ratio": "16:9"
  }'
```

```bash 图生视频(Pro 单图)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "viduq3-pro",
    "prompt": "人物缓缓转身,镜头推近",
    "image_urls": ["https://cdn.example.com/portrait.jpg"],
    "duration": 5,
    "resolution": "1080p"
```

## 通用参数

<ParamField body="model" type="string" required>
  模型 ID,见 [可用模型](#可用模型) 表
</ParamField>

<ParamField body="action" type="string">
  操作类型,Pro 档 **可省略**(按 `image_urls` 数量自动路由),其他档建议显式指定。可选值:
  - `generate` — 文生视频(仅 Pro)
  - `image2video` — 图生视频
  - `reference` — 参考生视频(标准 / Mix / Turbo 默认)
  - `first_last_frame` — 首尾帧(仅 Pro)
</ParamField>

<ParamField body="prompt" type="string" required>
  视频描述文本,建议描述 **动作 / 镜头 / 氛围**,人物外观由 `image_urls` 决定
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  画面宽高比,可选值:
  - `16:9` — 横版宽屏
  - `9:16` — 竖版长屏
  - `4:3` — 横版
  - `3:4` — 竖版
  - `1:1` — 正方形
</ParamField>

<ParamField body="resolution" type="string" default="720p">
  输出分辨率,各模型支持档位见 [可用模型](#可用模型) 表
</ParamField>

<ParamField body="duration" type="integer" default="5">
  视频时长(秒),范围 **1-16** 任意整数。Mix 档支持 1 秒短视频
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图片 URL 数组,**1-7 张**:
  - 标准 / Mix / Turbo:作为主体 / 风格参考(R2V)
  - Pro 档:0 张文生 / 1 张图生 / 2 张首尾帧,详见 [自动路由](#自动路由说明)
</ParamField>

<ParamField body="first_frame_image" type="string">
  视频首帧图 URL,**Pro 档 `first_last_frame` 模式必填**
</ParamField>

<ParamField body="last_frame_image" type="string">
  视频尾帧图 URL,**Pro 档 `first_last_frame` 模式必填**
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [回调机制](/cn/docs/sync-async#异步任务-webhook)
</ParamField>

<ParamField body="callback_events" type="string[]">
  订阅的回调事件,默认订阅终态(`succeeded` / `failed` / `cancelled`)
</ParamField>

## 自动路由说明

`viduq3-pro` 按 `image_urls` 数组长度自动选择生成模式,**通常无需显式传 `action`**:

| `image_urls` 数量 | 路由模式 | 等价 `action` | 说明 |
|---|---|---|---|
| `0` 张(或不传) | **文生视频** | `generate` | 纯 prompt 驱动 |
| `1` 张 | **图生视频** | `image2video` | 以图为起始画面 |
| `2` 张 | **首尾帧** | `first_last_frame` | 第一张作首帧,第二张作尾帧 |

<Note>
显式传 `first_frame_image` + `last_frame_image` 时优先采用,会覆盖 `image_urls` 的首尾帧路由。
</Note>

其他档(`viduq3` / `viduq3-mix` / `viduq3-turbo`)默认走 **R2V 参考生** 模式,`image_urls` 1-7 张作为主体 / 风格参考。

## 资源限制

| 项目 | 限制 |
|---|---|
| 参考图片 | 最多 7 张,单张 ≤ 30MB,JPG / PNG / WEBP |
| 首尾帧图 | JPG / PNG / WEBP,需公网可下载链接 |
| Prompt 长度 | 建议 ≤ 2000 字符 |
| 视频时长 | 1-16 秒任意整数(Mix 支持 1 秒短视频) |
| 输出 | MP4,链接 24 小时有效 |
