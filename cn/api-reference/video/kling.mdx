快手可灵 Kling 视频生成系列。

**四代模型分工**:

- **kling-v2.6** — 经典稳定版,标准模式 720P 无声 / 专业模式 1080P 可有声,语义和动态稳定性进一步打磨
- **kling-v3** — 新一代基础模型,新增 **4K 模式**,文生时长可拓展到 **15 秒**,原生支持有声视频
- **kling-v3-omni** — v3 系列**统一多模态接口**,文生 / 图生 / 视频参考全走同一 endpoint,prompt 支持 `<<<image_N>>>` 引用语法
- **kling-video-o1** — 首款**推理增强**视频模型,生成前对提示词与参考素材深度规划,物理一致性、复杂运动、长动作语义跟随上表现最优

按 **分辨率 / 模式** 计费,单位 `$/秒`。

## 调用示例

<CodeGroup>

```bash 文生视频(最简)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v3",
    "prompt": "一只柴犬穿宇航服在月球上行走,电影感光影",
    "duration": 5,
    "resolution": "1080p",
    "aspect_ratio": "16:9",
    "mode": "pro",
    "audio": true
  }'
```

```bash 图生视频(首帧)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v2.6",
    "action": "image2video",
    "prompt": "人物缓缓走向镜头,微风吹动头发",
    "first_frame_image": "https://cdn.example.com/portrait.jpg",
    "duration": 5,
    "resolution": "1080p",
    "mode": "pro"
  }'
```

```bash Omni 多模态参考
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v3-omni",
    "action": "reference",
    "prompt": "把 <<<image_1>>> 中的角色放到 <<<image_2>>> 的森林场景里奔跑",
    "image_urls": [
      "https://cdn.example.com/char.jpg",
      "https://cdn.example.com/forest.jpg"
    ],
    "duration": 6,
    "resolution": "1080p",
    "aspect_ratio": "16:9"
  }'
```

```bash O1 推理增强(视频参考)
curl -X POST https://www.qingbo.dev/v1/tasks \
  -H "Authorization: Bearer $WAVE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-video-o1",
    "action": "reference_video",
    "prompt": "保持 <<<image_1>>> 角色形象,沿用参考视频的镜头运动",
    "image_urls": ["https://cdn.example.com/char.jpg"],
    "video_list": [
      {
        "video_url": "https://cdn.example.com/ref.mp4",
        "refer_type": "feature",
        "keep_original_sound": "no"
      }
    ],
    "duration": 10,
    "resolution": "1080p"
  }'
```

## 通用参数

<ParamField body="model" type="string" required>
  模型 ID,见 [可用模型](#可用模型) 表
</ParamField>

<ParamField body="action" type="string" default="generate">
  操作类型,可选值:
  - `generate` — 文生视频(T2V)
  - `image2video` — 图生视频,需配合 `first_frame_image` 或 `image_urls`
  - `first_last_frame` — 首尾帧补全,需 `first_frame_image` + `last_frame_image`(v2.6 / v3 / o1)
  - `reference` — 多模态参考生视频,需配合 `image_urls`(omni / o1)
  - `reference_video` — 视频参考生视频,需配合 `video_list`(omni / o1)
</ParamField>

<ParamField body="prompt" type="string" required>
  视频描述文本。Omni / O1 中可用 `<<<image_N>>>` 语法引用 `image_urls` 第 N 张图(N 从 1 开始)
</ParamField>

<ParamField body="n" type="integer" default="1">
  一次生成几条候选,通常为 `1`
</ParamField>

<ParamField body="seed" type="integer" default="-1">
  随机种子,`-1` 为随机;固定值可复现相似结果
</ParamField>

<ParamField body="aspect_ratio" type="string" default="16:9">
  画面宽高比,**仅 T2V 生效**;图 / 视频参考模式跟随源素材比例。可选值:
  - `16:9` — 横版宽屏
  - `9:16` — 竖版长屏
  - `1:1` — 正方形
</ParamField>

<ParamField body="resolution" type="string" default="720p">
  输出分辨率,**与 `mode` 联动**。各模型支持范围见 [可用模型](#可用模型):
  - `720p`
  - `1080p`
  - `4K`(仅 v3 / omni)
</ParamField>

<ParamField body="duration" type="integer" default="5">
  视频时长(秒)。v2.6 / o1 仅 `5` 或 `10`;v3 / omni 支持 3-15 任意整数
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图片 URL 数组。**单元素数组**触发 I2V;多张用于 Omni / O1 的多模态参考(可被 `<<<image_N>>>` 引用)
</ParamField>

<ParamField body="first_frame_image" type="string">
  视频首帧图 URL。配合 `image2video` 或 `first_last_frame` action 使用(v2.6 / v3 / o1)
</ParamField>

<ParamField body="last_frame_image" type="string">
  视频尾帧图 URL,配合 `first_last_frame` action 使用
</ParamField>

<ParamField body="video_urls" type="string[]">
  参考视频 URL 数组(单元素)。Omni / O1 简化形态,等价于挂一段 `refer_type=feature` 的 `video_list`
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [回调机制](/cn/docs/sync-async#异步任务-webhook)
</ParamField>

<ParamField body="callback_events" type="string[]">
  需要推送的事件类型,默认终态全推
</ParamField>

## 特殊语法说明

**`<<<image_N>>>` 提示词图片引用**(Omni / O1 适用)

在 `prompt` 中用 `<<<image_N>>>` 占位符显式引用 `image_urls` 数组中的第 N 张图(N 从 **1** 开始)。模型会把占位符替换为对应图像内容,用于:

- 角色一致性 — `<<<image_1>>>` 锁定主体形象
- 场景拼合 — `<<<image_1>>>` 中的角色出现在 `<<<image_2>>>` 的环境
- 多素材引导 — 任意位置嵌入 N 张参考图

```text 示例
"把 <<<image_1>>> 中穿红衣的女孩,放到 <<<image_2>>> 的雪山场景里,镜头缓慢推进"
```

对应 `image_urls`:

```json
"image_urls": [
  "https://cdn.example.com/girl-red.jpg",
  "https://cdn.example.com/snow-mountain.jpg"
]
```

<Note>
非 Omni / O1 的 v2.6 / v3 不解析 `<<<image_N>>>` 占位符,会作为字面文本送入模型。
</Note>

## 资源限制

| 项目 | 限制 |
|---|---|
| 参考图片(单张) | ≤ 30MB,JPG / PNG / WEBP |
| 参考图片(数量) | I2V:1 张;Omni / O1 多模态:建议 ≤ 4 张 |
| 参考视频(`video_list`) | MP4 / MOV,≤ 100MB,2-30 秒,**最多 1 段** |
| 提示词 | ≤ 2500 字符 |
| 输出 | MP4,链接 24 小时有效 |
