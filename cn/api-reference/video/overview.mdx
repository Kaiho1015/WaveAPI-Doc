---

WaveAPI 支持多家视频生成模型,**统一通过异步任务接口** `/v1/tasks` 调用。视频生成通常耗时 10 秒到几分钟不等,提交后返回 `task_id`,通过轮询或 Webhook 回调获取结果。

## 请求端点

```
POST https://www.qingbo.dev/v1/tasks
```

异步流程详见 [任务系统](/cn/docs/task-system) — Submit → Status → Result + 可选 webhook。

## 通用参数(全系列共有)

> 各模型实际支持范围不同,**详见各 vendor 文档**。

<ParamField body="model" type="string" required>
  模型 ID(group_name),从上面的 vendor 文档选择
</ParamField>

<ParamField body="action" type="string" default="generate">
  操作类型,可选值取决于模型,常见:
  - `generate` — 文生视频(T2V)
  - `image2video` — 图生视频(I2V)
  - `first_last_frame` — 首尾帧
  - `reference` / `reference_video` — 参考生视频(R2V)
  - `edit` / `video_edit` / `style_transfer` — 视频编辑
  - `video_continuation` / `extend` — 视频续写
</ParamField>

<ParamField body="prompt" type="string">
  视频描述文本,T2V 必填;其他模式可选作引导
</ParamField>

<ParamField body="duration" type="integer">
  视频时长(秒),范围取决于模型
</ParamField>

<ParamField body="aspect_ratio" type="string">
  画面宽高比,如 `16:9` / `9:16` / `1:1` 等
</ParamField>

<ParamField body="resolution" type="string">
  输出分辨率,如 `720p` / `1080p` / `4K`
</ParamField>

<ParamField body="image_urls" type="string[]">
  参考图片 URL 数组(I2V / R2V 用)
</ParamField>

<ParamField body="first_frame_image" type="string">
  视频首帧图 URL(首尾帧模式用)
</ParamField>

<ParamField body="last_frame_image" type="string">
  视频尾帧图 URL,需配合 `first_frame_image`
</ParamField>

<ParamField body="video_urls" type="string[]">
  参考视频 URL 数组,**单元素即可**(视频续写 / R2V 视频参考 / 视频编辑用)
</ParamField>

<ParamField body="audio_urls" type="string[]">
  参考音频 URL 数组,**单元素即可**(驱动音 / 自定义配音用)
</ParamField>

<ParamField body="callback_url" type="string">
  Webhook 回调地址,任务终态时调用。详见 [任务系统](/cn/docs/task-system#webhook)
</ParamField>

## 提交响应示例

```json
{
  "task_id": "task-wave1775285160b950328499",
  "model": "veo3.1-quality-official",
  "action": "generate",
  "status": "queued",
  "created_at": 1775285160040,
  "progress": 0
}
```

拿到 `task_id` 后调 [`GET /v1/tasks/{task_id}`](/cn/api-reference/task/status) 轮询,直到 `status = completed` 取视频 URL。

## 生成模式速查

| 模式 | 触发字段 | 典型 vendor |
|---|---|---|
| **文生视频(T2V)** | 仅 `prompt` | 全部支持 generate action 的 vendor |
| **图生视频(I2V)** | `+ first_frame_image` 或 `image_urls`(单图) | Seedance / Kling / Hailuo / Vidu Pro / Wan / Veo / Sky / Grok |
| **首尾帧** | `first_frame_image + last_frame_image` | Seedance Pro Quality / 1.5 / 2.0 / Kling / Veo / Vidu Pro / Sky |
| **参考生视频(R2V)** | `image_urls`(多图,1-9 张) | Seedance / Kling Omni / Vidu / Sky / HappyHorse |
| **视频参考(R2V 视频)** | `+ video_urls`(参考视频) | Seedance 2.0 / Kling Omni / Sky / Wan |
| **视频续写** | `video_urls`(单段) | Wan(其他档暂未上架)/ Sky |
| **视频编辑(EDIT)** | `video_urls + prompt`(可加 `image_urls` 作风格参考) | Wan VideoEdit / HappyHorse / Sky |
| **音频参考** | `+ audio_urls` | HappyHorse / Wan / Sky / Seedance 2.0 |

<Note>
并非所有模型都支持全部模式 — 各 vendor 实际支持的 `action` 列表 + 字段范围请查看对应文档。后端会校验请求字段是否在 vendor 声明的能力范围内,不在范围会返回错误。
</Note>

## 字段命名规范

- **媒体引用统一复数** — 一律 `image_urls` / `video_urls` / `audio_urls`,即使单视频也用单元素数组 `["one.mp4"]`
- **画面比例统一** `aspect_ratio` — vendor 内部用 `size` / `ratio` 等是实现细节,无需感知
- **分辨率统一** `resolution` — vendor 内部用 `mode` / `quality` 等是实现细节
- **首尾帧字段带 `_image` 后缀** — `first_frame_image` / `last_frame_image`(强调是图片资源)
