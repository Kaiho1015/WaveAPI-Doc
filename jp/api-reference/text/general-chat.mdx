---
title: "汎用チャットインターフェース"
api: "POST https://qingbo.dev/v1/chat/completions"
---

- 統一されたチャット API インターフェース、全ての主要テキスト生成モデルに対応
- OpenAI Chat Completions API フォーマットと完全互換
- `model` パラメータで異なる AI モデルにシームレスに切り替え可能
- ストリーミング出力、関数呼び出しなどの高度な機能に対応

<RequestExample>
```bash cURL
curl -X POST https://qingbo.dev/v1/chat/completions \
```

```python Python
from openai import OpenAI

client = OpenAI(
    base_url="https://qingbo.dev/v1",
    api_key="YOUR_API_KEY"
)

response = client.chat.completions.create(
    model="gpt-5",
    messages=[
        {"role": "system", "content": "你是一个专业的AI助手。"},
        {"role": "user", "content": "解释一下冒泡排序算法。"}
    ],
    temperature=0.7,
    max_tokens=1024
)

print(response.choices[0].message.content)
```

```go Go
    if err != nil {
        panic(err)

    defer resp.Body.Close()

}
```

```java Java
import java.net.URI;

            .build();
```

## Authorizations

<ParamField header="Authorization" type="string" required>
  全てのインターフェースで Bearer Token 認証が必要です

  API Key の取得：

  [API Key 管理ページ](https://qingbo.dev/dashboard/keys) にアクセスして API Key を取得してください

  リクエストヘッダーに以下を追加してください：

  ```
  Authorization: Bearer YOUR_API_KEY
  ```
</ParamField>

## Body

<ParamField body="model" type="string" required>
  モデル名

  対応モデル：

  * **OpenAI**: `gpt-5`, `gpt-5-chat-latest`, `gpt-5-mini`, `gpt-5-nano`, `gpt-5-pro`
  * **Anthropic**: `claude-sonnet-4-5-20250929`, `claude-opus-4-1-20250805`, `claude-haiku-4-5-20251001`, `claude-opus-4-1-20250805-thinking`, `claude-sonnet-4-5-20250929-thinking`
  * **Google**: `gemini-2.5-pro`, `gemini-2.5-flash`, `gemini-2.5-pro-thinking`, `gemini-2.5-flash-lite`,`gemini-3-pro-preview-thinking`,`gemini-3-pro-image-preview`,`gemini-3-pro-preview`
  * **DeepSeek**: `deepseek-v3.1-250821`, `deepseek-v3.1-think-250821`, `deepseek-v3-0324`
  * **Doubao**: `doubao-seed-1-6-251015`, `doubao-seed-1-6-flash-250828`, `doubao-seed-1-6-thinking-250715`
  * **X.AI**: `grok-4.1`
  * その他のモデルも随時追加中...
</ParamField>

<ParamField body="messages" type="array" required>
  会話メッセージリスト

  <Expandable title="メッセージオブジェクトの属性">
    <ParamField body="role" type="string" required>
      ロールタイプ：`system`（システムプロンプト）、`user`（ユーザー入力）、`assistant`（モデル応答）
    </ParamField>

    <ParamField body="content" type="string" required>
      メッセージ内容、文字列またはマルチモーダルコンテンツ配列に対応
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="temperature" type="number">
  出力のランダム性を制御、範囲 0-2

  * 低い値（例：0.2）で出力がより確定的になります
  * 高い値（例：1.8）で出力がよりランダムになります

  デフォルト値：1.0
</ParamField>

<ParamField body="max_tokens" type="integer">
  生成する最大 Token 数

  モデルごとに最大値の制限が異なります。各モデルのドキュメントを参照してください
</ParamField>

<ParamField body="stream" type="boolean">
  ストリーミング出力を使用するかどうか

  * `true`: ストリーミング返却（SSE 形式）
  * `false`: 完全なレスポンスを一括返却

  デフォルト値：false
</ParamField>

<ParamField body="top_p" type="number">
  核サンプリングパラメータ、範囲 0-1

  生成テキストの多様性を制御します。temperature との併用ではなく、どちらか一方の使用を推奨します

  デフォルト値：1.0
</ParamField>

<ParamField body="frequency_penalty" type="number">
  頻度ペナルティ、範囲 -2.0 から 2.0

  正の値を設定すると、同じ語彙の繰り返し使用が抑制されます

  デフォルト値：0
</ParamField>

<ParamField body="presence_penalty" type="number">
  存在ペナルティ、範囲 -2.0 から 2.0

  正の値を設定すると、新しいトピックについて話す可能性が高まります

  デフォルト値：0
</ParamField>

<ParamField body="stop" type="string or array">
  停止シーケンス

  最大4つのシーケンスを指定でき、これらのシーケンスに遭遇すると生成が停止します
</ParamField>

<ParamField body="n" type="integer">
  生成する応答の数

  デフォルト値：1
</ParamField>

## Response

<ResponseField name="id" type="string">
  レスポンスの一意識別子
</ResponseField>

<ResponseField name="object" type="string">
  オブジェクトタイプ、固定値 `chat.completion`
</ResponseField>

<ResponseField name="created" type="integer">
  作成タイムスタンプ
</ResponseField>

<ResponseField name="model" type="string">
  実際に使用されたモデル名
</ResponseField>

<ResponseField name="choices" type="array">
  生成された応答リスト

  <Expandable title="属性">
    <ResponseField name="index" type="integer">
      選択肢のインデックス
    </ResponseField>

    <ResponseField name="message" type="object">
      メッセージ内容

      <Expandable title="属性">
        <ResponseField name="role" type="string">
          ロールタイプ（assistant）
        </ResponseField>

        <ResponseField name="content" type="string">
          生成されたテキスト内容
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="logprobs" type="object">
      対数確率情報（デフォルトは null）
    </ResponseField>

    <ResponseField name="finish_reason" type="string">
      終了理由

      可能な値：

      * `stop` - 自然終了
      * `length` - 最大長に到達
      * `content_filter` - コンテンツフィルター
      * `function_call` - 関数呼び出し
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="usage" type="object">
  Token 使用統計

  <Expandable title="属性">
    <ResponseField name="prompt_tokens" type="integer">
      入力メッセージの Token 数
    </ResponseField>

    <ResponseField name="completion_tokens" type="integer">
      生成コンテンツの Token 数
    </ResponseField>

    <ResponseField name="total_tokens" type="integer">
      合計 Token 数
    </ResponseField>

    <ResponseField name="prompt_tokens_details" type="object">
      入力 Token の詳細情報

      <Expandable title="属性">
        <ResponseField name="cached_tokens" type="integer">
          キャッシュされた Token 数
        </ResponseField>

        <ResponseField name="text_tokens" type="integer">
          テキスト Token 数
        </ResponseField>

        <ResponseField name="audio_tokens" type="integer">
          音声 Token 数
        </ResponseField>

        <ResponseField name="image_tokens" type="integer">
          画像 Token 数
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="completion_tokens_details" type="object">
      生成 Token の詳細情報

      <Expandable title="属性">
        <ResponseField name="text_tokens" type="integer">
          テキスト Token 数
        </ResponseField>

        <ResponseField name="audio_tokens" type="integer">
          音声 Token 数
        </ResponseField>

        <ResponseField name="reasoning_tokens" type="integer">
          推論 Token 数（思考モデル専用）
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="system_fingerprint" type="string">
  システムフィンガープリント識別子（バックエンド設定の追跡用）
</ResponseField>
