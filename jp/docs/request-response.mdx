## リクエスト形式

全てのリクエストは JSON 形式を使用し、以下の headers が必要です：

```bash
Content-Type: application/json
Authorization: Bearer YOUR_API_KEY
```

## 成功レスポンス

エンドポイントによってレスポンス構造は異なりますが、それぞれの API 仕様に準拠しています：

- **Chat Completions** — OpenAI 標準レスポンス形式
- **Claude Messages** — Anthropic 標準レスポンス形式
- **Gemini** — Google 標準レスポンス形式
- **非同期タスク** — WaveAPI 統一タスクレスポンス形式

## エラーレスポンス

```json
{
  "error": {
    "type": "invalid_request_error",
    "code": "invalid_request",
    "message": "具体的なエラー説明メッセージ"
  }
}
```

## エラーコード

| ステータスコード | 説明 | 対処方法 |
|--------|------|----------|
| 400 | リクエストパラメータが無効 | リクエストパラメータの形式を確認してください |
| 401 | 認証失敗 | API Key が正しいか確認してください |
| 402 | 残高不足 | アカウントにチャージしてください |
| 429 | リクエスト頻度超過 | リクエスト頻度を下げてください |
| 500 | サーバー内部エラー | しばらく後に再試行してください |
